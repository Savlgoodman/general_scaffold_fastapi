# General Scaffold Backend - FastAPI

基于 FastAPI 的后台管理系统脚手架，集成完整 RBAC 权限系统，支持用户管理、角色权限、菜单管理、API 日志、操作审计、异常监控、通知公告等功能。开箱即用，前端通过 OpenAPI Schema 自动生成调用代码，实现无感对接。

## 快速开始

### 方式一：Docker 开发（推荐）

**Windows:**
```powershell
.\start-docker-dev.ps1
```

**Linux/Mac:**
```bash
chmod +x start-docker-dev.sh
./start-docker-dev.sh
```

启动后访问：
- **应用**: http://localhost:8000
- **API文档**: http://localhost:8000/docs
- **默认账号**: admin / admin123

### 方式二：本地开发

```bash
# 1. 安装依赖
pip install -r requirements.txt

# 2. 配置数据库（修改 config.yaml）

# 3. 启动应用（会自动建表）
python -m uvicorn main:app --host 0.0.0.0 --port 27001 --reload
```

## 技术栈

- **FastAPI**: Web 框架
- **SQLAlchemy**: ORM（同步模式）
- **Alembic**: 数据库迁移管理
- **PostgreSQL**: 关系型数据库
- **Redis**: 缓存和会话存储
- **JWT**: 用户认证
- **Pydantic**: 数据验证
- **Loguru**: 日志管理
- **bcrypt**: 密码加密

## 项目结构

```
app/
├── api/admin/            # 路由层（全部挂在 /api/admin/ 下）
│   ├── auth.py           # 认证（登录、注册、刷新Token）
│   ├── admin_users.py    # 用户管理
│   ├── roles.py          # 角色管理
│   ├── permissions.py    # 权限管理
│   ├── menus.py          # 菜单管理
│   ├── api_logs.py       # API 请求日志
│   ├── error_logs.py     # 系统异常日志
│   ├── login_logs.py     # 登录日志
│   ├── operation_logs.py  # 操作审计日志
│   ├── notices.py        # 通知公告
│   └── system_info.py    # 系统监控
├── core/                 # 核心模块（config、database、redis、logger、app）
├── middleware/           # 中间件（认证、API日志）
├── models/              # SQLAlchemy 模型（全部继承 Base，含软删除）
├── schemas/             # Pydantic Schema（请求/响应校验）
├── services/            # 业务逻辑层
alembic/                 # 数据库迁移
docs/                    # 项目文档
config.yaml              # 配置文件（DEV/TEST/PROD 三环境）
```

## 功能特性

### 1. 用户认证
- ✅ JWT 登录 / 注册 / Token 刷新
- ✅ 密码 bcrypt 加密
- ✅ 图形验证码（EasyCaptcha）
- ✅ 登录失败计数与锁定

### 2. RBAC 权限系统
- ✅ 用户 → 角色 → 权限（标准 RBAC 三层结构）
- ✅ 用户 → 角色 → 菜单（菜单可见性控制）
- ✅ 用户级权限覆盖（per-user grant/deny，绕过角色分配）
- ✅ 权限匹配支持通配符（如 `/api/admin/users/detail/*`）
- ✅ 角色权限继承（effect: allow/deny + 优先级）
- ✅ 动态权限同步（从 OpenAPI spec 自动同步权限列表）

### 3. 系统监控
- ✅ 系统资源监控（CPU、内存、磁盘）
- ✅ Redis 连接状态
- ✅ 数据库连接状态
- ✅ JWT Token 状态

### 4. 日志体系（四大日志）
- ✅ **API 请求日志** — 所有接口的调用记录（路径、方法、参数、响应码、耗时、用户/IP）
- ✅ **登录日志** — 登录时间、IP、User-Agent、成败结果
- ✅ **操作审计日志** — 谁在什么时候做了什么操作（增删改），记录操作前后数据
- ✅ **系统异常日志** — WARNING/ERROR/CRITICAL 异常自动入库，含堆栈、请求上下文

### 5. 通知公告
- ✅ 公告 CRUD（标题、内容、状态：草稿/已发布/已撤回）
- ✅ 置顶与排序
- ✅ 发布时间管理

### 6. 数据库
- ✅ Alembic 迁移（所有表结构变更走迁移脚本）
- ✅ 软删除（所有模型有 is_deleted 字段，不做物理删除）
- ✅ 统一响应格式（`Response(code, message, data)`）

### 7. 安全
- ✅ 全局异常捕获，堆栈不泄露到前端
- ✅ CORS 配置（支持多域名）
- ✅ 超级管理员角色（绕过权限校验）

## 数据库迁移

```bash
# 生成迁移
alembic revision --autogenerate -m "描述"

# 应用迁移
alembic upgrade head

# 回滚
alembic downgrade -1
```

详细文档：[docs/database-migration.md](docs/database-migration.md)

## 配置说明

`config.yaml` 支持 DEV / TEST / PROD 三个环境，环境变量可覆盖。关键配置项：

- `database` — PostgreSQL 连接
- `redis` — Redis 连接
- `jwt` — Token 密钥和过期时间
- `logging` — 日志级别和文件轮转
- `api_log` — 是否记录请求体/响应体
- `cors` — 跨域白名单
